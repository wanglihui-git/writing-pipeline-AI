# 写作 Agent 技术方案（最终版）

## 1. 文档信息
- 文档名称：写作 Agent 技术方案
- 对应 PRD：`docs/产品需求文档-写作Agent.md`
- 版本：v1.0（MVP 可开发版）
- 语言：中文

## 2. 目标与范围
本方案用于指导 MVP 开发，覆盖从语料导入、风格建模、先大纲后正文生成、评分、改写到飞书 Bot 交互的端到端实现。

### 2.1 目标
- 支持基于“简略版提纲 + 指定作者风格”生成 5000~10000 字长文。
- 强制执行“先大纲后正文”流程。
- 支持整篇与局部多轮改写。
- 支持自动评分 + 人工评分记录。
- 单任务在常规负载下控制在 10 分钟内完成。

### 2.2 不在 MVP 范围
- 多用户权限系统
- 私有化离线部署
- 多作者融合生成（放到后续版本）

## 3. 总体架构
采用“飞书长链接事件入口 + FastAPI 服务层 + 本地数据与索引 + 可配置千问模型调用”的分层架构。

### 3.1 架构组件
- 飞书 Bot 接入层（Python 飞书官方 SDK，长链接模式）
- 应用服务层（FastAPI）
- 任务处理层（应用内同步函数调用）
- 数据层（SQLite + 本地文件）
- 检索层（Chroma 向量库 + 风格特征索引）
- 模型层（千问，多模型可配置）

### 3.2 数据流概览
1. 飞书消息进入长链接事件循环。  
2. 事件解析后调用 FastAPI 任务接口。  
3. API/Bot 进程内同步执行（大纲、正文、评分、改写）。  
4. 生成过程读写本地 `data/`，并调用 Chroma/SQLite。  
5. 阶段结果回写飞书会话（进度推送 + 结果摘要）。  

## 4. 技术选型与决策
### 4.1 服务层
- 语言与框架：Python + FastAPI
- 原因：生态成熟、与 NLP/LLM 工具链兼容性好、后续可平滑接 Web UI。

### 4.2 飞书接入
- 模式：飞书官方 SDK 的长链接（Long Connection）事件接收。
- 说明：不采用传统 Webhook 回调作为主通道。
- 能力要求：
  - 支持命令解析与会话追踪
  - 支持长任务进度推送
  - 支持任务失败回执与重试提示

### 4.3 存储与索引
- 元数据：SQLite（任务、版本、评分、作者、配置快照）
- 向量检索：Chroma（本地持久化）
- 文本与中间产物：本地 `data/` 目录

### 4.4 执行模式
- 模式：同步 API 执行（无消息队列）
- 策略：创建任务后由 API 直接执行大纲/正文/评分流程；改写与语料入库同样同步执行。
- 结果：接口返回即代表本次处理已完成（或失败），任务状态可通过查询接口追踪。

### 4.5 模型调用
- 模型提供方：千问
- 规则：不同环节可用不同模型，且统一通过配置文件切换。

建议目录结构如下（MVP）：

## 6. 语料处理与双索引设计（重点）
现代与小众作者拟合能力的核心在于“召回准 + 风格稳”。采用“双粒度切块 + 双索引召回”。

### 6.1 清洗与切块
- 输入：`data/raw/<author>/*.txt`
- 清洗步骤：
  - 编码统一（UTF-8）
  - 去除异常符号与重复空行
  - 去重（近似重复段去重）
  - 标题与正文分离
- 双粒度切块：
  - 段落块：保留原生语气与句法节奏
  - 滑窗块：400~800 字窗口，窗口重叠 80~120 字

### 6.2 双索引
- 语义索引（Chroma）：
  - 存 embedding 与文本 chunk
  - 用于按主题召回候选片段
- 风格特征索引（SQLite）：
  - 每个 chunk 一行特征记录
  - 典型字段：
    - `avg_sentence_len`
    - `long_short_sentence_ratio`
    - `punctuation_profile`
    - `transition_word_density`
    - `rhetoric_density`
    - `first_person_ratio`
    - `assertiveness_score`

### 6.3 检索策略
1. 基于用户提纲抽取主题查询。  
2. Chroma 召回 TopK（建议 40）。  
3. 使用风格画像进行风格重排（style rerank）。  
4. 选出 TopN（建议 8~12）作为风格锚点输入。  
5. 仅注入“风格特征与短引用摘要”，避免原文复现。  

## 7. 固定 Pipeline 设计
严格执行，不提供跳过大纲确认的分支。

### 7.1 阶段 P0：请求标准化
- 输入：选题、切入角度、核心命题、论证框架、叙事骨架、目标读者、字数、风格参数
- 输出：结构化请求 JSON（写入 DB）

### 7.2 阶段 P1：大纲生成
- 读取风格锚点与主题证据
- 生成“章节目标 + 段落目的 + 证据挂载点”的大纲
- 状态变更：`WAIT_OUTLINE_CONFIRM`

### 7.3 阶段 P2：大纲确认
- 用户确认/修改
- 仅在确认后进入正文生成

### 7.4 阶段 P3：正文生成
- 按章节分段生成（支持 2~4 并发）
- 每章生成后执行局部一致性检查
- 章节拼接后执行全局润色（术语统一、逻辑衔接、风格校准）

### 7.5 阶段 P4：评分
- 规则分 + LLM 裁判分 + 综合分
- 生成评分卡并落库

### 7.6 阶段 P5：改写
- 整篇改写或局部改写
- 局部改写自动回写上下文衔接段
- 重新评分与版本化存储

## 8. 改写系统实现
### 8.1 改写类型
- 全文改写：语气、信息密度、叙事风格调整
- 局部改写：章节/段落/句群精修

### 8.2 关键约束
- 支持 `keep_facts=true`：事实不变，仅调整表达
- 局部改写后自动执行上下文衔接重写，防止语气断裂
- 每次改写创建新版本，不覆盖旧版本

### 8.3 建议接口
- `POST /tasks/{task_id}/rewrite/full`
- `POST /tasks/{task_id}/rewrite/partial`
- `GET /tasks/{task_id}/versions`

## 9. 评分体系实现（三层）
### 9.1 层一：规则特征分（可解释）
- 风格特征匹配度（词汇、句法、修辞、节奏）
- 结构完整率（提纲覆盖、逻辑连续、首尾完整）
- 自然度特征（模板句、空话密度、机械连接词）

### 9.2 层二：LLM 裁判分
- 使用独立评分提示词，不与生成提示词复用
- 输出：风格、结构、自然度三分项 + 证据说明

### 9.3 层三：综合分
- 加权策略（固定）：
  - 风格相似度：40%
  - 结构完整率：35%
  - 自然度：25%
- 不区分名家/小众作者阈值（MVP 统一标准）

## 10. 飞书 Bot 交互设计（长链接）
### 10.1 指令建议
- `/outline`
- `/generate`
- `/rewrite_full`
- `/rewrite_part`
- `/score`
- `/versions`

### 10.2 会话与状态
- 以 `task_id` 绑定一次完整写作任务
- 同一会话可持续迭代同一任务版本

### 10.3 进度推送
- 长任务按阶段推送状态：
  - 已接收
  - 语料检索中
  - 大纲生成中
  - 等待确认
  - 正文生成中（附百分比）
  - 评分中
  - 完成/失败

## 11. 配置与可变参数
### 11.1 模型配置（示例）
`config/models.yaml` 建议：
- `outline_model`
- `draft_model`
- `polish_model`
- `judge_model`
- `embedding_model`

### 11.2 运行配置（示例）
`config/app.yaml` 建议：
- 默认字数、并发度、超时阈值
- Chroma 路径、SQLite 路径
- 重试次数、退避策略
- 飞书应用配置（app_id、app_secret、事件开关）

## 12. 性能与可靠性方案
### 12.1 性能策略
- 章节级并行生成（2~4 并发）
- 检索与生成分离，避免重复 embedding
- 分段缓存，减少重复调用

### 12.2 降级策略（已接受）
- 高负载降级为低并发或串行
- 优先保证任务成功完成与结果质量

### 12.3 稳定性策略
- 失败自动重试（指数退避）
- 中间产物持久化（至少保留大纲）
- 幂等任务提交（防重复触发）

## 13. 数据库与核心表（SQLite）
建议核心表：
- `authors`：作者基本信息
- `documents`：原始文本与清洗文本映射
- `chunks`：切块元数据
- `style_features`：风格特征
- `tasks`：任务主表（状态机）
- `task_versions`：大纲/正文/改写版本
- `scores`：评分结果
- `feedback`：人工评分与备注

## 14. 状态机设计
建议任务状态：
- `RECEIVED`
- `OUTLINE_GENERATING`
- `WAIT_OUTLINE_CONFIRM`
- `DRAFT_GENERATING`
- `SCORING`
- `READY`
- `REWRITING`
- `FAILED`

状态机约束：
- 未进入 `WAIT_OUTLINE_CONFIRM` 前禁止正文生成。
- 任意失败状态允许基于最近可用版本重试。

## 15. 接口清单（FastAPI）
### 15.1 任务接口
- `POST /tasks`：创建任务（含简略提纲）
- `GET /tasks/{task_id}`：查询状态
- `POST /tasks/{task_id}/outline/confirm`：确认大纲
- `POST /tasks/{task_id}/generate`：触发正文生成（内部校验必须已确认大纲）

### 15.2 改写与评分接口
- `POST /tasks/{task_id}/rewrite/full`
- `POST /tasks/{task_id}/rewrite/partial`
- `POST /tasks/{task_id}/score`
- `GET /tasks/{task_id}/versions`

### 15.3 语料接口
- `POST /authors/{author}/ingest`
- `POST /authors/{author}/reindex`
- `GET /authors/{author}/profile`

## 16. 安全与合规边界
- 明确禁止逐句复现原文、禁止请求“复刻某篇具体文章”。
- 风格锚点只做特征约束与短引用摘要，不输出大段原文拼接。
- 输出结果附带系统声明：为风格化原创生成，不代表原作者观点。

## 17. 可观测性与运维
- 日志维度：
  - 请求日志
  - 模型调用日志（时延、token、错误码）
  - 阶段耗时日志
  - 评分与人工反馈日志
- 监控指标：
  - 任务成功率
  - 平均完成时长
  - 大纲确认后一次成稿率
  - 平均人工“像”评分

## 18. MVP 验收映射
与 PRD 验收项对应：
- 人工“像”评分 >= 4/5
- AI 味不浓（人工可接受）
- 长文结构完整率 >= 90%
- 端到端时长 <= 10 分钟（常规负载）

## 19. 实施里程碑（建议）
- M1（第 1 周）：项目骨架、飞书长链接接入、任务状态机
- M2（第 2 周）：语料清洗、切块、Chroma 建库、风格特征提取
- M3（第 3 周）：固定 Pipeline（大纲 -> 正文 -> 评分）
- M4（第 4 周）：改写系统、版本管理、进度推送、稳定性优化
- M5（第 5 周）：验收测试与参数调优

## 20. 后续扩展（V1.1+）
- 可视化界面（任务面板、版本对比、局部编辑）
- 多候选稿并行生成 + 自动融合
- 作者画像可视化（雷达图与趋势图）
